# Lessons Learned

## Day 1
- Create .gitignore before any config files to avoid accidentally committing secrets
- Resource group naming: rg-{project}-{component} is readable and sortable in the portal
- ADO Scrum PBI = User Story — same concept, different name
- GitHub CLI (gh) is significantly faster than using the web UI for repo operations
- Always verify az account show before creating resources — wrong subscription is a common mistake
